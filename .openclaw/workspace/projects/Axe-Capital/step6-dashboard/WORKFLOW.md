# Axe Capital Dashboard — Guide & Workflow

This dashboard is a **static** Vite + React UI that reads pre-generated artifacts from `step6-dashboard/public/`.

No live APIs yet. The “pipeline” is file-based.

---

## 1) What the dashboard shows (Panels 1–7)

### Panel 1 — Portfolio State
**Reads:** `public/portfolio.json`

Generated by Step 5 portfolio tracker.

Shows:
- NAV, positions value, cash, unrealized P&L
- per-position table (with stop level + distance)
- sector weights
- “Positions requiring attention” (RED/YELLOW)

### Panel 2 — Watchlist & Targets
**Reads:**
- `public/portfolio.json`
- `public/targets.json`

Purpose:
- Per-position rules for **profit targets** and **stops**.

Targets/stops can be defined either as absolute prices or as % from the base price.
- Base price is **avg cost**, falling back to **last price**.

Edit rules here:
- `step6-dashboard/public/targets.json`

### Panel 3 — Risk Snapshot
**Reads:**
- `public/portfolio.json`
- `public/targets.json`

Computes an approximate stop-loss risk model:
- per-position risk ≈ `max(last - stop, 0) * shares`
- total risk $ = sum of per-position risk
- risk % NAV = total risk / NAV

Stop selection:
- first: `targets.json` → `stop_price`
- fallback: `portfolio.json` → `stop_loss_level`

### Panel 4 — Research Digest
**Reads:** `public/weekly-review-latest.json`

Shows “weekly review” highlights:
- portfolio vs SPY comparison
- top sector concentration rows

This file comes from Step 5 reports.

### Panel 5 — Decision Log
**Reads:** `public/decision-log.jsonl`

A simple audit trail, appended by automation.

### Panel 6 — Automation
**Reads (presence checks):**
- `public/portfolio.json`
- `public/targets.json`
- `public/weekly-review-latest.json`
- `public/approval-queue.md`
- `public/decision-log.jsonl`
- `public/health.json`

If an artifact is missing, this panel will show **MISSING**.

### Panel 7 — Runbook
Currently placeholder links.

## 2) The work pipeline (Phase 1: file-based)

### Source of truth inputs
- **IBKR CSV export** (raw):
  - `projects/Axe-Capital/00_Dashboard/data/raw/portfolio-current.csv`
- **Rules:**
  - `projects/Axe-Capital/step6-dashboard/public/targets.json`
- **Investor profile knobs:**
  - `projects/Axe-Capital/INVESTOR_PROFILE.md` (hishtalmut room parsing)

### Step 5 (portfolio tracking)
Location:
- `projects/Axe-Capital/step5-portfolio-tracking/`

Command (via venv):
- `projects/Axe-Capital/.venv/bin/python -m axe_portfolio.cli`

Outputs:
- normalized CSV: `00_Dashboard/data/portfolio_latest.normalized.csv`
- weekly review JSON: `projects/Axe-Capital/reports/weekly-review-latest.json`
- dashboard portfolio artifact: `projects/Axe-Capital/step6-dashboard/public/portfolio.json`

### Step 6 (dashboard)
Location:
- `projects/Axe-Capital/step6-dashboard/`

Runs:
- `npm run dev` (local dev)
- `npm run build` (production build)

Reads artifacts from:
- `projects/Axe-Capital/step6-dashboard/public/`

### Step 7 (automation)
Location:
- `projects/Axe-Capital/step7-automation/`

Entry script:
- `run_daily_refresh.sh`

What it does:
1) appends `decision-log.jsonl` (“refresh started”)
2) runs Step 5 tracker to regenerate portfolio + weekly review
3) syncs weekly review into dashboard `public/weekly-review-latest.json`
4) writes `public/health.json`
5) appends `decision-log.jsonl` (“refresh complete”)

---

## 3) How to operate it (daily loop)

1) Update raw broker export
- replace: `00_Dashboard/data/raw/portfolio-current.csv`

2) Update targets/stops if needed
- edit: `step6-dashboard/public/targets.json`

3) Run daily refresh
- `projects/Axe-Capital/step7-automation/run_daily_refresh.sh`

4) Open dashboard
- `cd projects/Axe-Capital/step6-dashboard && npm run dev`

---

## 4) Known limitations (current phase)

- No backend, no live prices beyond what’s in the CSV.
- “Approval queue” is a placeholder file right now.
- Runbook links are placeholders.
- Risk model is approximate (no slippage, assumes stop fills).
